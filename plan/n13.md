# N13 — Issues-as-home: node schema + intake

NODE N13 deps=N11 done=issue-node-schema.sh lane=agent :: Define the issue-as-node schema (one issue = one node; body = intent + Depends-on: edges + done-condition) and the intake that mints a node-issue

## Intent

Make a **GitHub issue the durable form of a plan-node**. The schema (proven by
`criteria/issue-node-schema.sh` against a fixture / the API): one issue = one node; the **body**
carries intent + `Depends-on: #<n>` edges + the done-condition; exactly one `status:*` label names
the lifecycle lane (the machine state N14 guards); `ws:`-style stream/axis labels stay separate
(axis-separation, borrowed from chiron). Intake mints a node-issue from a converged brief.

deps=N11 (the ruling). This node defines the **states and the node format** that N14's guard
enforces — so N14 depends on it. It is the schema half; N14 is the enforcement half.
