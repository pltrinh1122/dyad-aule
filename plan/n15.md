# N15 — Reverse the mirror: issues → local cache

NODE N15 deps=N14 done=cache-mirror.sh lane=agent :: Reverse N9's mirror — sync issues (truth) DOWN into a local plan cache for the rare offline case + local ./check; reuse the _board engine inverted

## Intent

Flip the N9 mirror's direction. N9 was files(truth)→issues(view). Now it is
**issues(truth)→local-cache(view/offline)**: CI (and a `d-start` seam) sync the node-issues down
into a local, git-tracked cache under a cache home, so the practice keeps working offline (the rare
case the Operator scoped) and so `./check` has a local ground to read plan-shape from. The cache is
**derived and disposable** — never a second writer; drift between cache and issues is surfaced by
the same tripwire as N14. Much of `bin/_board` (manifest/plan-ops/reconcile) is reusable, inverted.

deps=N14 (the cache must reflect guarded, legal status). Unlocks nothing further; it is the offline
half of the ruling ("offline operates like a cache").
