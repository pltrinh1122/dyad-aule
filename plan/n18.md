NODE N18 deps=N14,N17 done=d-land.sh lane=agent :: The d-land operator disposition directive — chat, closes dispose→execute (releases the node for execution); metered as disposition-turns/node; built to deprecate into standing authorization; grounded in dyad-chiron, adapted to aule

## Intent

The Operator's **primary disposition channel** (chat; issue-comments secondary). `d-land <node>`
records the `dispose → execute` transition (guarded by N14): an agent-lane node is released for
execution (WIP=1 — I branch and work it; the open PR carrying the `Node:` trailer is the derived
"executing" ground); an operator-lane node's disposition *is* its content. On a non-disposed node,
d-land is a warn-not-halt urgency override (chiron's #16). Metered from day one (disposition-turns
per node) and **built to deprecate** into standing per-stream authorization as execution trust earns
(the Operator's veto always stands). A `d-*` operator discipline (O8); ships with `bin/d-land` +
`criteria/d-land.sh`.

deps=N14 (the dispose→execute transition it closes must be guard-enforced) and N17 (d-sense must
converge a node to `dispose` before d-land can release it). The last node of the flip.
