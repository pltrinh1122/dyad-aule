# N10 — Decompose plan/ into self-contained per-node files

NODE N10 deps=N0 done=plan-grain.sh lane=agent :: Decompose plan/ into self-contained per-node files (plan/<id>.md) + a derived DAG index; the plan is a collection of nodes, not a monolith

## Intent

Reshape the `plan/` kind-home so the plan **is the collection of self-contained work nodes**, not
one monolithic file: each node lives in its own `plan/<id>.md` (intent + edges + done-condition +
design notes + provenance, co-located), and the DAG/frontier is **derived** (`bin/_frontier
--all`), never stored as an authored table. `plan/close-the-gaps.md` remains as the plan-level
**contract/index** (how-to-read, pick-order, the cross-cutting decision records). Landed WITH its
mechanism (`criteria/plan-grain.sh`) — never as an asserted-but-unwired invariant (the N8 rule).

## Why per-node files (Operator probe, 2026-07-07 — the decision record)

Probed: the "why the nodes are not GitHub issues" decision defended **repo-vs-external**; it never
defended **monolith-vs-per-node**. Those are independent axes, and the second was an un-argued
default from N0. Decided: **decompose** — the plan is a collection of self-contained nodes.

- **Zero fidelity cost.** Every argument that killed gh-issues (single-writer, derived-not-stored
  state, merge-gate, `./check`-reachable, equal durability) holds identically for per-node *files*
  as for one file. None required a *single* file. Truth stays repo-grounded and merge-gated.
- **The lived friction argued for it.** N8 and N9 each needed a large "design notes" block
  appended far below their one-line NODE entry — a node's spec was split across the monolith (an
  O2 violation at the node grain). A `plan/<id>.md` co-locates the whole node. Form from friction,
  not ahead of it — and the friction landed.
- **More faithful to this dyad's own ethos.** "State is derived, never stored" is the core rule.
  The monolith *stored* the DAG as an authored artifact; per-node files + a **derived** index
  (`bin/_frontier --all`) make the graph a view computed from atomic ground.
- **The deriver already supported it.** `bin/_frontier` and `bin/_dyad-rt` already glob
  `plan/*.md` and grep `^NODE`/`^DISPOSITION` — so the contract just generalizes "this file" →
  "these files"; migration cost was near-zero.
- **Concurrency (the deferred G8) becomes conflict-free** — parallel node work touches different
  files, never colliding on one — and **path-derived provenance sharpens** (a `Node: N10` commit
  touching `plan/n10.md` carries its id in the path; browsing `plan/` *is* a rudimentary board).
- **What the monolith won, kept:** DAG topology at a glance and atomic multi-node replans survive
  as a *derived* view (`bin/_frontier --all`) over atomic files, plus the plan-level contract in
  `close-the-gaps.md`. The better of both.

## The grain invariant (enforced by `criteria/plan-grain.sh`)

- Exactly **one `NODE` line per `plan/<id>.md`**, and the file basename matches that node's id
  (case-insensitive): `plan/n9.md` ⟺ `NODE N9 …`.
- The contract file `plan/close-the-gaps.md` holds **zero `NODE` lines** — it is meta, not a node.
- A disposition node's `DISPOSITION <id>:` line lives in that node's own file.
